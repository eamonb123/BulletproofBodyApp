# Product Model — Bulletproof Body

> Status: Active source of truth
> Last updated: 2026-03-08

## Mission
Make fat loss feel easy, non-judgmental, and immediate for busy people by showing high-impact swaps inside foods they already eat.

## Core Promise
Same life, smarter choices, visible progress.

- Keep foods they love.
- Lower calories without feeling punished.
- Increase satiety where possible (protein, volume, fiber).
- Turn every choice into a clear, rewarding game loop.

## Primary User
Career-driven adult under cognitive load (meetings, family, low bandwidth) who wants results without meal-prep perfection.

They need:
- Fast clarity, not nutrition lectures.
- Emotional safety (no shame language).
- One useful decision now, not a full lifestyle overhaul first.

## Product Principles
1. Clarity over complexity.
2. Evidence over motivation.
3. Permission over restriction.
4. Progress over perfection.
5. Memory over one-off interaction.

## First-Win Path (90 Seconds)
1. Pick a familiar food/restaurant.
2. See current order vs smarter swap side-by-side.
3. See calories saved and weekly impact.
4. Save to personal plan.
5. Feel momentum and continue.

## Interaction Contract
- Never shame. Never moralize food.
- Keep copy short and actionable.
- Every screen should answer: “What should I do next?”
- Every tap should give feedback (visual number change, saved state, or reveal).

## Personalization + Memory Loop
The system should remember:
- Preferred foods/cravings.
- Accepted swaps.
- Frequency and context (when/why eaten).
- Weekly impact over time.

The system should show memory back to user:
- “Your plan” language.
- Saved rows and frequency controls.
- Cumulative impact metrics.

## Guardrails
- Do not force “clean eating” replacements that break craving integrity.
- Do not hide key numbers behind dense text.
- Do not ship UI that requires squinting or heavy reading under stress.

## Build Priority
1. First-win flow quality (clarity + speed + emotional safety)
2. Readability and UX consistency
3. Personalization and memory reliability
4. Catalog expansion and richer media
